Swift5.0入门-数据类型

换macbook笔记本之后，发现之前很多在windows上很多好用的工具在mac下没有好的替代品，遂萌生了学习swift来自己编写mac工具的想法，后续便是自己学习swift5.0的笔记。

Swift中的数据类型

变量常量的声明

  1. 在swift中，通过var关键字来声明变量

    var 变量名称: 类型 = 初始值

  2. 通过let来声明常量：

    let 常量名称: 类型 = 值

数据类型

整数

  1. swift 提供了8，16，32和64位编码的有符号和无符号整数
  2. 命名方式： 例如8位无符号整数的类型是UInt8，32位有符号整数的类型是Int32
  3. 通过min和max属性可以访问每种整数类型的最小值和最大值
  4. Swift提供了一个额外的整数类型：Int，它拥有与当前平台的原生长度相同的长度
  5. 同事Swift也提供了UInt类型，来表示平台长度相关的无符号整数
  6. 建议在用到整数的地方都是用Int

浮点类型

  1. Double: 64位浮点数，至少有15位数字的精度
  2. Float： 32位浮点数，至少有6位数字的精度
  3. 在两种类型都可以使用的情况下，推荐使用Double类型.

Bool

  • Bool: true和false
  • Swift的类型安全机制会阻止你用一个非布尔量的值替换掉Bool

类型别名

  • 类型别名是一个为已存在的类型定义的一个可选择的名字
  • 用关键typealias定义一个类型的别名
  • 当你想通过在一个在上下文中看起来更适合可具有表达性的名字来引用一个已存在的类型时，这是别名就非常有用

Tuple(元组)

  • 元组把多个值合并成单一的复合型的值
  • 元组内的值可以是任何类型，而且可以不必是统一类型
  • 元组中的每一个元素可以指定对应的元素名称
  • 如果没有指定名称的元素也可以使用下表的方式来引用

Tuple的修改

  • 用var定义的元组就是可变元组，let定义的元组就是不可变元组

  • 不管是可变还是不可变元组，元组在创建后就不能增加和删除元素个数

  • 可以对可变元组的元素进行修改，但是不能改变其类型

  • any类型可以修改为任何类型

Tuple分解

  • 将一个元组的内容分解成单独的常量或变量
  • 如果只需要使用其中一部分数据，不需要的数据可以用下划线(_)代替

Tuple作为函数返回值

  • 使用Tuple为函数返回多个值
  • 返回值的Tuple可以在函数声明的返回类型处被命名

Optional

为什么需要Optional

  • Object-C里的nil是无类型的指针：不知道nil究竟是Int还是String
  • Object-C里的数组、字典、集合等不允许放入nil：有时候需要放入nil来做占位符
  • Object-C所有的对象变量都可以为nil
  • Object-C只能用在对象上，而在其他地方又用其他的特殊值(例如NSNotFound)表示缺失值

Optional定义

  • 通过在变量类型后面添加?定义，表示这里有值或者根本没有值
  • 通过给可选变量赋值一个nil来将它设置为没有值
    • 在Swift中，nil不是指针，他是值缺失的一种特殊类型，任何类型的可选项都可以设置成nil而不仅仅是对象类型

Optional使用

Optional-if语句和强制展开使用
  • 可选项没法直接使用
  • 需要用!展开之后才能使用(意思是我知道这个可选项里面有值，展开吧)
Optional强制展开
  • 使用!来获取一个不存在的可选值会导致运行错误，在使用!强制展开之前必须确保可选项中包含一个非nil值
Optional绑定
  • 使用可选项绑定来判断可选项是否包含值，如果包含就把值赋给一个临时的常量或者变量
  • 可选绑定可以与if或者while的语句使用来检查可选项内部的值，并赋值给一个变量或常量
  • 同一个if语句中包含可选项绑定，用逗号分割即可。如果任一可选项结果是nil或者布尔值为false，那么整个if判断会被看作false
Optional-隐式展开
  • 有些可选项一旦被设定值之后，就会一直有值，在这种情况下，可以去掉检查的必要，也不必每次访问的时候都进行展开
  • 通过在声明的类型后面添加一个!来书写隐式展开可选项
  • 隐式展开可选项主要被用在Swift类的初始化过程中
Optional-可选链(传递性)
  • 可选项后面加问号
  • 如果可选项不为nil,返回一个可选项结果，否则返回nil

字面量

  • 字符串字面量是被双引号(“)包裹的固定顺序文本字符
