memcached常用命令

memcached交互协议主要有文本协议和二进制协议，其中二进制协议会比文本协议略快。文本协议主要包含在执行item的命令过程中，其中item命令有两行，第一行:$$\langle command name \rangle \langle key \rangle \langle flags \rangle \langle exptime \rangle \langle bytes \rangle $$

第二行：value，存储的值
常用命令包括数据存储、读取等命令：

数据存储

set:数据存在时更新，数据不存在存储

add:当数据不存在时存储

replace:当数据存在时替换之前的数据存储

数据读取

get:key可以多个，多个key之间空格

gets:与get相同，但是会多返回一个数字，判断数据是否被修改过，相当于版本控制

cas:check and set 多线程中的同步操作。主要用于多个客户端同时读取的事后保证原子操作。当最后一个参数与gets返回的数字一致时才存储，否则返回EXISTS。

数字的加减

incr:增加

decr:减少

追加与清除命令

append:追加到已经存在的数据之后，当存在时才会存储

prepend:追加到已经存在的数据之前，当存在时才会存储

delete:删除缓存数据，数据存在时，删除成功返回DELETED，不存在时返回NOT_FOUND

flush_all:将当前所有缓存数据设置为过期，不释放内存.

其他命令

stats:返回memcached服务器状态信息

返回的字段中部分解释:

名称 含义
pid server端的pid
uptime 自启动到现在的时间(秒)
time 当前服务器时间
version 版本号
libevent 当前memcached服务器依赖的libevent版本
pointer_size 指针大小
curr_items 当前已经存储的item数目
rusage_user 该进程累计的用户时间，单位：秒
rusage_system 该进程累计的系统时间，单位：秒
curr_items Memcached 当前存储的内容数量
total_items 服务端从启动到现在总共存储的item数量
bytes 存储item花费的内存大小
curr_connections 当前客户端连接数量
total_connections 服务端从启动到现在总共的连接数
connection_structures 分配的链接数
cmd_get get次数
cmd_set set的次数
get_his get命中次数
get_misses get未命中次数
evictions 为新的item释放内存空间而被移除的有效item的数目。如果cache的size比较小，则淘汰策略经常发生
bytes_read 从cache中读取的总字节数
bytes_written 写入cache的总字节数
limit_maxbytes server分配的最大内存数量

stats items:查看items行

stats cachedump slabs_id limit_num和上面的stats items结合查看具体的item命令:

slabs_id:上面stats items返回的STAT items后面的数字
limit_num:返回几条记录，0:所有记录

stats slabs:返回各个slab的信息:chunk的大小、数目、使用情况

stats sizes:所有的item的大小和个数

stats reset 清空所有统计数据

stats系列命令能够为我们清晰的提供的memcached服务器的运行状态，我们可以通过这些命令组合去监控分布式memcached服务器的运行情况，及时的处理问题和提出需要优化的地方等。
